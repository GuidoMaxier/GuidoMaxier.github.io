Defensa del Proyecto de Programación

Introducción:

La aplicación que hemos desarrollado tiene como objetivo proporcionar a los usuarios una plataforma para la gestión de servidores y canales de chat. A continuación, presentamos las funcionalidades clave de la aplicación y destacamos los desafíos y limitaciones que enfrentamos durante su desarrollo.

Funcionalidades:

  1. Crear usuarios: Los usuarios pueden registrarse proporcionando su nombre, contraseña y otros datos necesarios. La imagen de perfil se puede elegir después de la creación del usuario.
  2. Iniciar sesión: Los usuarios pueden iniciar sesión. Si no existen, se les sugiere registrarse.
  3. Pantalla Principal:
    • Primera columna: Muestra un listado de los servidores a los que el usuario pertenece. Si no pertenece a ninguno, se muestra un mensaje indicando esto. Se puede crear un servidor nuevo.
    • Segunda columna: Muestra un listado de los canales del servidor seleccionado. Si no hay canales, se muestra un mensaje indicándolo. Se puede crear un canal nuevo.
    • Tercera columna: Muestra mensajes ordenados cronológicamente en el chat del canal seleccionado. Se puede escribir un nuevo mensaje.
  4. Los mensajes solo pueden ser modificados o eliminados por el usuario que los ha creado.
  5. Perfil de Usuario: Los usuarios pueden actualizar sus datos personales, incluyendo la imagen de perfil.
  6. Manejadores de Errores Personalizados: Implementamos manejo de errores para códigos 400, 404, 403 y 500 para mejorar la experiencia del usuario.
  7. Buscador de Servidores: Los usuarios pueden buscar servidores por nombre. Los resultados muestran el nombre del servidor, la descripción (si la hay) y la cantidad de usuarios registrados.
  8. Administración de Sesión: Registramos la información de inicio de sesión y restringimos el acceso a usuarios logueados.
  9. Notificaciones e Invitaciones a un Servidor: Implementamos notificaciones e invitaciones para mejorar la comunicación entre usuarios.

Tecnologías Utilizadas:

  • Flask para el desarrollo del backend.
  • Python-mysql.connector para consultas a la base de datos MySQL.
  • HTML y CSS para la estructura y el diseño de la interfaz de usuario.
  • JavaScript para la interactividad en la interfaz de usuario.

Conclusiones:

Durante el desarrollo de este proyecto, enfrentamos varios desafíos, pero logramos implementar las funcionalidades clave requeridas. Aprendimos mucho sobre el diseño de aplicaciones web y la gestión de bases de datos. Planeamos continuar mejorando y completando las funcionalidades restantes en el futuro.
